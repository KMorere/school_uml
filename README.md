# TP UML : gestion d’une école, v1.0

### [à réaliser en groupe, un groupe pour chaque site]

L’expression des besoins du client est la suivante :
Notre école accueille des élèves qui suivent un ensemble de cours sur une certaine période, cours et période dépendant de la formation suivie. On ne souhaite pas, pour cette première version de l’application, ni gérer les différentes formations proposées ni les périodes pendant lesquelles elles se déroulent.
Chaque cours a une dénomination, s’étale entre deux dates et est confié à un enseignant, arrivé dans l’école à une certaine date.
On dispose également pour chaque élève ou enseignant de son prénom, son nom, son âge et son adresse, cette dernière étant constituée d’une rue, d’une ville et d’un code postal. On souhaite aussi pouvoir exploiter facilement l’ensemble de toutes les adresses (par ex. pour en connaitre la répartition géographique).
Chaque élève se voit en outre attribué un numéro d’élève unique.
L’objectif de cette première itération de l’application est d’avoir terminé la couche métier de votre application, et d’avoir validé l’ensemble de ses fonctionnalités, avant de passer à l’itération suivante lors de laquelle vous mettriez en oeuvre les couches DAO, puis Web !
Pour ce faire, vous devez réaliser plusieurs diagrammes UML (diagramme de cas d’utilisation, diagramme de classes, diagramme de séquence) ainsi que les spécifications fonctionnelles, avant de passer au développement.
NB : On ne vous demande pas de gérer la couche web ici (pas de connexion...), uniquement les opérations de gestion de l’école répondant aux besoins sous la forme d'une couche métier qu'il faudra tester avec une application.
